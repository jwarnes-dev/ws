Welcome to Willow

Helping teams build cohesive, efficent, and on-brand digital products

What is the Willow Design System?

Willow is a design system, a unified toolkit for scaling design and development across teams.

Design & branding guidance

Centralized standards for colors, typography, spacing, and layout

Reusable UI libraries

Ready-to-use Angular and React components built for Wellmark's needs

Unified documentation

A single source for design intent and implementation, with live code examples

Governance & contribution

Clear workflows to evolve the system through cross-functional input

Goals of Willow

Willow exists to make products teams faster, more consistent, and aligned

Consistency

Establish a unified visual language across all Wellmark applications

Efficiency

Reduce developer time by using prebuilt, tested components

Enablement

Make it easy for designers and engineers to apply system standards

Quality

Raise the bar for UI/UX across teams

Components of the
Willow Ecosystem

Willow brings tools and assets together to support design and development at scale

Components libraries

Angular and React packages built to Wellmark's standards

Figma

The source of truth for design - all patterns, tokens, and layouts start here

Reference website

Central hub for documentation, guidance, and live examples

Storybook

Interactive environment for teams to explore and test components

What Willow is not

Willow clarifies how teams build - not who builds or what gets built

Not staff augmentation

The Willow team will help product teams effectively utilize the design system. They cannot consult on product-specific UI Work.

Not a silver bullet

Willow will never contain everything a product UI requires. There will always be some unique UI work to suit product needs.

Not rigid

Developers are encouraged to build upon Willow components to suit product needs.

Not a "one and done" asset

Willow is continuously updated to evolve with our needs
